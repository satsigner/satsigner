# Standard (BIP39) vs Electrum seed derivation

Side-by-side comparison of how mnemonic phrases are turned into keys in standard wallets vs Electrum.

| Step                         | Standard (BIP39 / BIP32)                                                                                                                        | Electrum                                                                                                                                        |
| ---------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------- |
| **1. Input**                 | Mnemonic (12/15/18/21/24 words) + optional passphrase                                                                                           | Same (often 12 words) + optional passphrase                                                                                                     |
| **2. Normalization**         | Mnemonic and passphrase in NFKD (Unicode)                                                                                                       | Mnemonic: NFKD, lowercased, trimmed, single spaces                                                                                              |
| **3. Seed derivation**       | **PBKDF2-HMAC-SHA512**<br>• Password: NFKD(mnemonic)<br>• Salt: `"mnemonic"` + NFKD(passphrase)<br>• Iterations: 2048<br>• Output: 64-byte seed | **PBKDF2-HMAC-SHA512**<br>• Password: NFKD(mnemonic)<br>• Salt: `"electrum"` + NFKD(passphrase)<br>• Iterations: 2048<br>• Output: 64-byte seed |
| **4. Version / type**        | None (same process for all)                                                                                                                     | **HMAC-SHA512**(key = `"Seed version"`, msg = normalized mnemonic) → hex prefix: `01` = standard, `100` = segwit, `101` = 2FA                   |
| **5. Master key**            | BIP32: seed → master extended key (network-specific)                                                                                            | Seed → BIP32-style master (path/usage differ)                                                                                                   |
| **6. Derivation path**       | BIP32 paths, e.g. `m/84'/0'/0'` (BIP84), `m/44'/0'/0'` (BIP44)                                                                                  | **Standard / 2FA:** `m`<br>**Segwit:** `m/0'` (or `m/0h`)                                                                                       |
| **7. Script / address type** | Chosen by path (e.g. BIP84 → P2WPKH, BIP44 → P2PKH)                                                                                             | **Standard / 2FA:** P2PKH<br>**Segwit:** P2WPKH                                                                                                 |
| **8. Result**                | Same mnemonic (+ passphrase) → same seed → same keys on any BIP39/BIP32 wallet                                                                  | Same mnemonic (+ passphrase) → **different** seed and keys than BIP39; only matches Electrum-compatible wallets                                 |

## Takeaway

Both use PBKDF2 to stretch mnemonic + passphrase into a seed, but **different salt** and **no “seed version” step** in BIP39, plus **different paths and script types**. The same 12 words therefore produce **different keys** in a standard wallet vs Electrum; they are not interchangeable.
